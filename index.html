Comandos GIT

$ git --help
Configurar usuário:
$ git config --global user.name "Alexandre Hesse"
Configurar email:
$ git config --global user.email "ale.hesse@gmail.com"
Verifica as configurações de nome e email:
$ git config user.name
$ git config user.email
Mostra o path:
$ pwd
Limpar tela:
Ctrl + L
Criar pasta:
$ mkdir suapasta
Entrar pasta:
$ cd suapasta
Visualizar diretório:
$ ls
Visualização completa:
$ ls -l
Inicializar Git:
$ git init
Criar arquivo:
$ touch teste.txt
Status do Arquivo:
$ git status
Adicionar arquivo para o Stage (segundo estado, palco):
$ git add teste.txt
Retirar arquivo do Stage (palco):
$ git rm --cached seuarqivo
Realizar o Commit:
$ git commit -m "Primeiro Commit"
Visualizar os commit's:
$ git log
$ git log --oneline
obs: Para sair do pager less (END) e voltar ao terminal, você pode pressionar a tecla q.
Visualizar o conteúdo arq:
$ cat seuarquivo.extensão
Excluir arquivo:
$ git rm nomedoarquivo.extesao
$ git commit - "Arquivo Excluído"
Recuperar arquivo:
$ git log --diff-filter=D --summary
$ git checkout 7b10~1 arq2.txt (primeiros dig do commit + ~1 nomearquivo)
Visualizar alterações:
$ git diff
Reverter commit:
$ git revert xxxxx (inicio log do commit)
obs: Abre uma janela para exibir a reversão ... só fechar
Remover arq do Stage(palco)
$ git rm --cached seuarqivo
$ git reset seuarquivo
$ git reset
obs: remove todos os arquivos do Stage (palco)
$ git reset --hard origin/main (Volta no último Pull da brench)
$ git reset --hard xxxxx(volta até o log indicado)
obs:ATENÇÃO --hard apaga tudo!!! stage, dirertório e Commit
REVERT X RESET
Revert(Mais seguro - Cria um novo commit com as alterações)
Reset(Remove as alterações do stage e diretório)
Deletar projeto:
$ cd .. (para sair da pasta do projeto)
$ pwd (verificar o diretório)
$ rm -rf pastaDoProjeto
