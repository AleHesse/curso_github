GIT BRANCH

Branches são as ramificações de um projeto. No GitHub, é utilizado para separar e gerenciar as versões do projeto. A branch principal é denominada "main".

O objetivo da new_branch é permitir trabalhar no projeto sem alterar a branch principal.

Criando uma nova branch:

No Terminal:

$ git branch

Obs: O * mostra em qual branch estamos trabalhando.

OBS: Sempre crie novas branches a partir da branch principal MAIN.

Criar branch:

$ git branch sua_new_branch

Criar branch e acessá-la diretamente:

$ git branch -b sua_new_branch

Obs: Antes de mudar de branch, realize qualquer commit pendente.

Trocar de branch:

$ git checkout sua_branch_escolhida

OBS: A partir de agora, podemos realizar alterações no projeto sem afetar a branch main (principal).

Deletar uma branch:

$ git branch -d sua_branch

OBS: Não se esqueça do commit ao mudar de branch!

GIT MERGE

O merge passa pela avaliação do code review (responsável pela aprovação).

Obs: Certifique-se de estar na branch MAIN para executar o merge.

Executar o merge:

$ git merge sua_branch

Obs: Caso você tenha criado uma branch que ficou desatualizada em relação à MAIN, acesse a sua branch e execute um merge a partir dela para a main para atualizá-la. Isso é muito comum em produção.

GIT STASH

O git stash permite que, ao estar trabalhando em uma branch, mesmo após várias alterações, você consiga voltar ao estado inicial.

Executar o stash:

$ git stash

Obs: O stash cria um hash, por exemplo: c35fe16, que você pode recuperar posteriormente.

Revertendo stash:

Acesse a branch que você estava e execute $ git stash list

Obs: O stash cria um índice ID, por exemplo:

    stash@{0}: WIP on myfirstbranch: ca4e8ed Criando mygithub.html
    stash@{1}: WIP on myfirstbranch: ca4e8ed Criando mygithub.html
    stash@{2}: WIP on myfirstbranch: ca4e8ed Criando mygithub.html

Para visualizar o conteúdo:

$ git stash show -p x (onde x é o índice desejado)

Para reverter:

$ git stash apply x (onde x é o índice desejado)
